Processes are global objects. Processes specify behavior, channels and global variables define the environment in which the processes run. The scope of a variable is global if it is declared outside all process declarations, and local if it is declared within a process declaration.

A condition can only be executed (passed) when it holds. If the condition does not hold, execution blocks until it does.

The semicolon is a statement separator (not a statement terminator, hence there is no semicolon after the last statement). Promela accepts two different statement separators: an arrow ->and the semicolon ;.

Run

A proctype definition only declares process behavior, it does not execute it. Initially, in the Promela model, just one process will be executed: a process of type init, that must be declared explicitly in every Promela specification

        init
        {       run A(); run B()
        }

run is used as a unary operator that takes the name of a process type (e.g. A). It is executable only if a process of the type specified can be instantiated. It is unexecutable if this cannot be done, for instance if too many processes are already running.

Active

In Promela, the active keyword is used to specify that a process is actively executing and not in a passive state. When a process is declared as active, it means that it is actively progressing through its code and can react to incoming messages or events. This is in contrast to passive processes, which are not actively executing and are waiting for some event to trigger their execution.

By declaring a process as active, the model designer can specify that the process should be ready to respond to events and messages, making the model more dynamic and realistic.

Atomic

By prefixing a sequence of statements enclosed in curly braces with the keyword atomic the user can indicate that the sequence is to be executed as one indivisible unit, non-interleaved with any other processes. It causes a run-time error if any statement, other than the first statement, blocks in an atomic sequence. This is how we can use atomic sequences to protect the concurrent access to the global variable state in the earlier example.

In other words

atomic sequences are blocks of code that are guaranteed to be executed without interruption by any other process. This means that once a process enters an atomic sequence, it will be allowed to run to completion without any other process being able to interfere with its execution.

Selection

    if
    :: (a != b) -> option1
    :: (a == b) -> option2
    fi

The selection structure contains two execution sequences, each preceded by a double colon. Only one sequence from the list will be executed. A sequence can be selected only if its first statement is executable. The first statement is therefore called a guard.
